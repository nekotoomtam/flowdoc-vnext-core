# Text-block v1 Producer Alignment

Status: Phase 250 accepted-write alignment.

## Outcome

Phase 250 aligns the runtime-owned table insertion producers with the target
Text-block v1 empty-block grammar. A newly inserted table cell owns one
paragraph text-block whose canonical inline children are `[]`.

The operation layer no longer creates an empty text leaf as a runtime caret
placeholder. Caret and visible placeholder behavior remain editor runtime
concerns and are not serialized into the authored document.

## Producer Audit

The source audit found two accepted-write producers that created
`{ type: "text", text: "" }`:

- `table.row.insert` created one empty paragraph text-block for each new cell;
- `table.column.insert` created one empty paragraph text-block for each row's
  new cell.

Both producers now use one operation-local empty paragraph factory. No other
runtime producer under `src` creates an empty text leaf.

## PASS

- Row and column insert preserve their existing graph, scope, invalidation,
  history, and revision-independent core operation contracts.
- Every generated cell still owns one concrete text-block.
- Generated text-blocks validate as `valid` under the target v1 grammar.
- Empty authored text-blocks contain no synthetic inline identity.

## FAIL / BLOCKER

- No blocker remains for accepted table insertion producers.
- Automatic package-read normalization remains blocked on the explicit
  document-version and migration decision.

## RISK

- Editor input mounting must create any caret host or placeholder as runtime
  state without persisting it as an empty text leaf.
- Other repositories or future producers can reintroduce empty text leaves if
  they bypass core operation contracts.

## UNKNOWN

- Phase 251 resolves target grammar enforcement to document v4.
- Phase 251 permits normalization only inside an explicit copy-forward
  migration or a future strict accepted-write preflight, never package read.

## Behavior Changed

New text-blocks generated by table row and column insertion now serialize
`children: []` instead of one empty text leaf.

## Intentionally Not Changed

- package v2/document v3 schema and parser;
- existing stored documents and fixtures;
- normalization on package read or write;
- text transaction and rich-inline operation semantics;
- editor caret, placeholder, selection, or DOM behavior;
- backend routes, revisions, and persistence;
- layout, fragments, pagination, and artifacts;
- image schema or asset lifecycle.
